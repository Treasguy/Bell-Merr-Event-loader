### How it works behind the scenes

When an authorized user logs into your site's custom backend portal (e.g., `[yourlocalpoliticsite.com/admin](https://yourlocalpoliticsite.com/admin)`), 

they fill out a form with the event title, date, location, and upload a banner image. When they click "Publish," the CMS doesn't save to a database; instead, **it uses an API to automatically commit a flat file (like `event-item.md` or a JSON block) and the image asset directly into your GitHub repository**.

Netlify detects the new file commit, automatically rebuilds your static site, and the new event appears live on the page within seconds. Wow.
